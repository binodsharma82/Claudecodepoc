## Technical Decisions and Rationale

### 1. React + Flask instead of extending FastAPI monolith

**Decision:** Build a separate React SPA + Flask API rather than extending `templates/index.html`.  
**Why:** The monolithic HTML/JS approach was already ~2000 lines and not maintainable. React enables component reuse, proper routing, and hot-reload DX. Flask was chosen over keeping FastAPI so the new backend could start cleanly without touching existing imports.  
**Trade-off:** Two Python processes in dev. Acceptable for a POC.

### 2. Vite proxy for `/api`

**Decision:** Vite dev server proxies `/api/*` to `http://127.0.0.1:5000` instead of configuring CORS in Flask.  
**Why:** Avoids CORS preflight complexity in dev. In production a single reverse proxy (nginx) would do the same.  
**Gotcha:** Node.js 17+ defaults to `::1` (IPv6) when you write `localhost`. Fixed by `host: '127.0.0.1'` in `vite.config.js`.

### 3. Pandas in-memory cache

**Decision:** Load all 10 Excel files once at startup into a `dict` of DataFrames; no database.  
**Why:** POC with static synthetic data тАФ no write operations needed. Avoids standing up SQLite/Postgres for a demo. Cache lives in process memory (~10MB), acceptable.  
**Limitation:** Restart Flask to refresh data.

### 4. AppContext for global filter state

**Decision:** Single React Context holds territory/period/BU selectors; all pages subscribe.  
**Why:** Filter state is cross-cutting тАФ every page needs the same territory/period context. Lifting state to Context avoids prop drilling across 7 pages. Redux would be overkill for a POC.

### 5. CSS custom properties for theming

**Decision:** All colours and layout dimensions as CSS vars in `index.css`.  
**Why:** Single source of truth for IQ2020 brand colours. Makes a future light-mode toggle trivial.

### 6. BU selector drives tab visibility

**Decision:** Each tab declares `bus: ['AIR','SBU','Vaccines']` and `TabNav` filters the list by active BU.  
**Why:** IQ2020 has distinct business units with different module sets. A Vaccines rep should not see AIR performance tabs.

### 7. GSK logo embedded as file asset

**Decision:** `frontend/src/assets/gsk1.png` imported in Header.jsx; original FastAPI dashboard still uses base64 in `templates/index.html`.  
**Why:** Vite handles image assets at build time (hashes, optimises). The base64 approach is fine for the static legacy HTML but not idiomatic for the React build pipeline.
